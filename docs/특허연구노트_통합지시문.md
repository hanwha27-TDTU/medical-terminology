# 특허용 연구노트 — 완전 통합 구현 지시문 (자매앱: 어학 마스터앱)

> **이 문서 하나로 처음부터 끝까지 구현한다.** 기본 연구노트 + 특허 전문가 검토 반영(전자서명·서버 타임스탬프·증거 필드·보고서) + 내보내기 노이즈 정리까지 전부 포함.
> **철학:** 과장 금지. "법적 보증"이 아니라 **"특허 검토용 구조화 증거 로그"**다. 자체생성 키는 CA 신원증명이 아니며, **외부 신뢰 기준점은 git 커밋 이력**임을 UI·문서에 계속 못 박는다.
> **발명자:** 사람 = **김부건 (Kim Bugeon) · 1982-08-18**. AI는 도구로서 보조. 이 귀속을 모든 기록에 자동으로 남긴다.
> **적용 순서:** Part 1(기반) → Part 2(전자서명·서버시각) → Part 3(증거 필드·보고서) → Part 4(내보내기 정리) → Part 5(RFC3161 공인 타임스탬프). 각 Part는 독립 커밋/PR, 버전 +0.01.

---

## 0. 설계 대원칙 (7가지 — 이걸 어기면 특허 증거로서 가치가 깎인다)

1. **원본 불변(append-only):** 한 번 저장한 기록은 절대 수정·삭제하지 않는다. 고칠 일이 생기면 **원본 id를 가리키는 새 정정 엔트리**를 추가한다.
2. **해시 체인:** 각 기록에 SHA-256 지문을 박고 직전 기록의 지문을 참조해 사슬로 잇는다. 중간을 바꾸면 재계산 지문이 달라져 드러난다.
3. **사람 vs AI 분리:** 착상·수정·최종결정 칸은 **사람(발명자)**의 것, 프롬프트·출력 칸은 **AI(도구)**의 것으로 구조를 나눠 기록한다.
4. **정직한 한계 표기:** 자동 로깅의 한계, 자체 키의 한계, "법적 보증 아님"을 UI에 계속 명시한다.
5. **백데이팅 금지:** 소급 기록도 작성일 = 지금. 날짜를 과거로 위조하지 않는다(권위 있는 실제 타임라인은 git 커밋).
6. **로컬 우선 + 클라우드 백업:** 오프라인에서도 동작(localStorage), 클라우드는 백업/동기화 수단.
7. **노이즈는 지우지 말고 뷰에서 필터:** append-only 데이터가 지저분해져도 삭제하지 않고 보기에서 거른다.

## 1.8 ⚠️ 멀티-앱 격리 — 전용 테이블 + 프로젝트 필터 (필수)

**여러 앱이 이 템플릿을 각자 쓰면, 같은 이름의 테이블 `research_notes`를 공유해 서로의 연구노트가 섞인다.** 각 엔트리에 `project_name`을 박아도 **읽을 때 거르지 않으면 무의미**하다. 오염은 화면·총 건수·내보내기·공개 해시 매니페스트, 그리고 **해시 체인 `previous_entry_hash`**(새 기록이 남의 엔트리를 물어 체인이 꼬임)까지 번진다. **처음부터 아래를 넣어라.**

**(1) 소유키 — 스탬프·필터 공용의 불변 상수.** `APP_INFO.name` 같은 가변값에 묶지 말 것(리네임 시 과거 자기 기록이 숨겨짐).
```js
const RN_PROJECT_NAME = '<이 앱 고유 이름>';           // 불변·고유. 과거/현재 엔트리의 project_name과 정확히 일치.
function _rnIsOwnProject(e){ return !!e && (e.project_name === RN_PROJECT_NAME || !e.project_name); } // 옛 무명 기록은 관대
```
`rnAddEntry`는 `project_name: RN_PROJECT_NAME`로 박는다.

**(2) `_rnLoad`를 단일 관문으로 필터** → 렌더·집계·검증·체인 prev·내보내기·매니페스트·제네시스·중복검사가 자동 격리. `rnPull` incoming도 같은 필터로 남의 행을 애초에 로컬에 안 쓴다.
```js
function _rnLoad(){ const a=safeJsonParse(localStorage.getItem(RN_STORAGE_KEY),[]); return Array.isArray(a)?a.filter(_rnIsOwnProject):[]; }
// rnPull: const incoming = (rows||[]).map(r=>r.data).filter(x=>x && x.id && _rnIsOwnProject(x));
```

**(3) 전용 테이블로 분리(권장·근본).** 테이블명을 앱 고유로(예 `research_notes_<앱>`). `research_notes`는 이전 공유 테이블로 두고 **1회만 읽어** 자기 프로젝트 행을 회수→전용 테이블로 이관(옛 테이블 **안 지움**·무손실). 완료 플래그는 기기별 localStorage. 전용 테이블 미생성 시 push 실패→플래그 미설정→다음 열람 재시도.

**(4) 연결 지점 전부 동반 수정(누락 방지):** `computeSchemaDrift`·`checkLiveSchema` 테이블명, 백업 실패 토스트 문구, 위 SQL DDL(전용 테이블명), **공개 매니페스트 CI 스크립트**(테이블명 + `project_name` 필터). 한 곳만 놓치면 "컬럼 없음" 오탐·매니페스트 오염 재발.

**(5) (선택) 옛 공유 테이블 정리 SQL — 복사 전용·가드형.** `project_name = 이 앱` **AND** `id가 전용 테이블에 존재`(`exists`) 2중 가드. 이관 전이면 0건, 남의 앱 행 불건드림. **자동 실행 금지** — 미리보기 count 후 사용자가 직접 실행.
```sql
-- 미리보기
select count(*) from public.research_notes r
 where r.data->>'project_name' = '<RN_PROJECT_NAME>'
   and exists (select 1 from public.research_notes_<앱> m where m.id = r.id);
-- 삭제(개수 확인 후에만)
delete from public.research_notes r
 where r.data->>'project_name' = '<RN_PROJECT_NAME>'
   and exists (select 1 from public.research_notes_<앱> m where m.id = r.id);
```

**불변성 존중:** 오염 시기 과거 엔트리의 `previous_entry_hash`가 숨겨진 남의 엔트리를 가리켜 검증 시 `chain_break`가 뜰 수 있으나 **소급 수정 금지**(엔트리 불변) — 격리 이후 새 기록부터 정상. 이 잔흔은 오염의 흔적으로 그대로 둔다.

> 두 앱이 공유 테이블을 만지므로 **순서**: ① 각 앱이 전용 테이블 먼저 생성 → ② 각 앱 1회 열어 이관 완료 → ③ 그 뒤에만 각자 정리 SQL. 가드(`exists`) 덕에 순서를 어겨도 0건이라 무손실이지만 이 순서가 가장 깔끔.

# PART 2 — 전자서명(ECDSA) + 서버 타임스탬프 (검토 ①②, 최우선)

## 2.1 서명 헬퍼

```js
const RN_KEY_JWK = 'langmaster_rn_ecdsa_key';   // ⚠️ 앱 고유 — 두 앱이 키를 공유하면 안 됨
function _rnB64(u8){ let s=''; for(let i=0;i<u8.length;i++) s+=String.fromCharCode(u8[i]); return btoa(s); }
function _rnB64ToBytes(b64){ const s=atob(b64); const u=new Uint8Array(s.length); for(let i=0;i<s.length;i++) u[i]=s.charCodeAt(i); return u; }
async function _rnGetKeypair() {
  if (typeof crypto==='undefined' || !crypto.subtle) return null;
  const stored=safeJsonParse(localStorage.getItem(RN_KEY_JWK), null);
  if (stored && stored.privateJwk && stored.publicJwk) {
    try { const priv=await crypto.subtle.importKey('jwk', stored.privateJwk, {name:'ECDSA',namedCurve:'P-256'}, false, ['sign']); return {priv, publicJwk:stored.publicJwk}; } catch(e){}
  }
  try {
    const kp=await crypto.subtle.generateKey({name:'ECDSA',namedCurve:'P-256'}, true, ['sign','verify']);
    const privateJwk=await crypto.subtle.exportKey('jwk', kp.privateKey);
    const publicJwk=await crypto.subtle.exportKey('jwk', kp.publicKey);
    safeSetItem(RN_KEY_JWK, JSON.stringify({privateJwk, publicJwk}));
    return {priv:kp.privateKey, publicJwk};
  } catch(e){ return null; }
}
async function _rnSign(hashHex) {
  const kp=await _rnGetKeypair(); if(!kp) return null;
  try { const sig=await crypto.subtle.sign({name:'ECDSA',hash:'SHA-256'}, kp.priv, new TextEncoder().encode(hashHex)); return {sig:_rnB64(new Uint8Array(sig)), pub:kp.publicJwk}; } catch(e){ return null; }
}
async function _rnVerifySig(entry) {
  if (!entry.entry_hash_signature || !entry.public_key_jwk) return null;  // 서명 없음
  try { const pub=await crypto.subtle.importKey('jwk', entry.public_key_jwk, {name:'ECDSA',namedCurve:'P-256'}, false, ['verify']);
    return await crypto.subtle.verify({name:'ECDSA',hash:'SHA-256'}, pub, _rnB64ToBytes(entry.entry_hash_signature), new TextEncoder().encode(entry.entry_hash)); } catch(e){ return false; }
}
```

# PART 5 — RFC3161 공인 타임스탬프(TSA) (증거력 ★★★★★)

> **무엇:** 제3자 신뢰기관(TSA)이 "이 해시(=이 내용)가 그 시각에 존재했음"을 **서명으로 증명**. 증거력 순위 `SHA-256(★★★) → Git Commit(★★★★) → RFC3161 TSA(★★★★★)`.
> **핵심 함정:** 브라우저는 TSA를 **직접 못 부른다** — ① TSA(DigiCert/GlobalSign/freeTSA)가 CORS 헤더를 안 줌 ② 요청/응답이 DER(ASN.1) 바이너리. → **Supabase Edge Function 1개로 중계**(Cloudinary 삭제 함수와 완전히 같은 패턴). TSA는 원본이 아니라 **해시만** 받는다(프라이버시 보존).

## 5.1 데이터 필드
- **해시 제외(사후 부착·가변):** `tsa_provider`, `tsa_request_hash`, `tsa_timestamp`, `tsa_token`, **`tsa_status`, `tsa_error`** → `_rnComputeHash` 제외 목록에 전부 추가(server_time·서명과 같은 부류).
- **해시 포함(생성 시 고정):** **`evidence_chain_id`** (§5.6 ⑥). 생성 시 정해져 불변이므로 오히려 해시에 넣어 변조를 막는다.

## 5.2 클라이언트 훅
```js
// 릴레이(Edge Function)가 배포 안 됐거나 Supabase 설정이 없으면 조용히 no-op(핵심 해시체인·서명 유지).
async function _rnRequestTsa(hashHex) {
  const cfg = getSupabaseConfig(); if (!cfg || !hashHex) return null;
  try {
    const res = await fetch(`${cfg.url}/functions/v1/rfc3161-timestamp`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json', 'apikey': cfg.key, 'Authorization': 'Bearer ' + cfg.key },
      body: JSON.stringify({ hash: hashHex, hashAlgo: 'sha256' }),
    });
    if (!res.ok) return null;                       // 미배포(404)·오류 → 조용히 건너뜀
    const j = await res.json().catch(() => null);
    if (!j || !j.token) return null;
    return { provider: j.provider || 'RFC3161 TSA', timestamp: j.timestamp || '', token: j.token }; // token=base64(TST)
  } catch (e) { return null; }
}
// 저장 시 자동 요청 → 성공하면 4필드 부착 후 재업서트. rnAddEntry 끝(서명·rnPushOne 뒤)에서 fire-and-forget 호출.
// _rnRequestTsa는 {ok, provider, timestamp, token} 또는 {ok:false, error}를 반환하게 한다(상태 저장용, §5.6 ⑤).
async function _rnAttachTsa(e) {
  if (!getSupabaseConfig() || e.tsa_token) return;
  const t = await _rnRequestTsa(e.entry_hash);
  const notes = _rnLoad(); const idx = notes.findIndex(n => n.id === e.id); if (idx < 0 || notes[idx].tsa_token) return;
  if (t.ok) { notes[idx].tsa_provider = t.provider; notes[idx].tsa_request_hash = e.entry_hash; notes[idx].tsa_timestamp = t.timestamp; notes[idx].tsa_token = t.token; notes[idx].tsa_status = 'success'; notes[idx].tsa_error = ''; }
```

## 5.3 표시 (카드 배지 + Markdown, 정직하게)
- 카드 배지 **상태별 구분**(§5.6 ⑤): `tsa_token` → `⏱️ 공인시각`(초록), `tsa_status==='failed'` → `⏱️ TSA실패`(빨강, title=사유), `'pending'` → `⏱️ TSA대기`(주황). + `evidence_chain_id` → `🔗 EC-…`.
- Markdown 무결성 섹션: `RFC3161 TSA: <provider> @ <timestamp> · status=<tsa_status>` + `tsa_token(base64 TST)` + **openssl 검증 명령 블록**(§5.6 ④). 헤더에 `Evidence Chain ID`.

## 5.4 Edge Function (Supabase에 배포 — 사용자 몫)
`supabase/functions/rfc3161-timestamp/index.ts` (Deno) + `config.toml`(`verify_jwt = true` — 앱이 anon 키를 Bearer로 보내 통과. 검증된 Cloudinary 함수와 동일).
함수가 하는 일: hex 해시 받기 → DER `TimeStampReq`(SHA-256 OID 고정, version=1, certReq=TRUE, ~59B) 생성 → `application/timestamp-query`로 `TSA_URL`(env, 기본 `https://freetsa.org/tsr`)에 POST → `TimeStampResp`에서 `TimeStampToken` 추출(base64) + genTime best-effort → `{provider, token, timestamp}` 반환. **전체 소스는 동봉 파일(index.ts) 사용.** 앱마다 프로젝트는 달라도 함수 코드는 동일.
- ★ **DER 바이트 계산·응답 파서는 반드시 Node로 사전 검증**: `TimeStampReq`가 정확히 59바이트인지, 토큰 슬라이스가 맞는지, genTime 정규식이 맞는지. (직접 짠 바이너리 코드는 검증 없이 안 넘긴다.)

## 5.5 정직한 한계
- 앱은 토큰을 **보관만** 한다(브라우저에서 CMS 전체 검증은 과함). **진짜 검증은 `openssl ts -verify`**(TSA CA 인증서 필요).
- **무료 TSA(freeTSA)는 신뢰기관 지속성 보장 없음** → 장기 보존이 중요하면 상업용(`TSA_URL` 교체) 권장. **단, 기술적 사실:** 이미 발급된 토큰은 기관이 문을 닫아도 **CA 인증서만 아카이브하면 계속 검증됨**(certReq=TRUE로 인증서 동봉). "서비스 중단 = 신규 발급만 불가"지 기존 증거 무효 아님. → **TSA CA 인증서를 반드시 별도 보관.**
- TSA는 "그 시각 그 해시 존재"만 증명. "누가"(발명자)는 서명·git·발명자 귀속으로 별도 보강.
- `data jsonb` 블롭이라 SQL 스키마 불변(새 컬럼 없음).

## 5.6 상태 관리 · Evidence Chain ID · openssl 명령 (특허 전문가 2차 검토 반영 — 강력 권장)
- **⑤② TSA 상태(`tsa_status`):** `pending`(저장 시·설정 있음) / `unavailable`(설정 없음) / `success` / `failed`(+`tsa_error` 사유). "실패=조용히 무시" 금지 — 어느 기록이 왜 안 됐는지 카드·MD에 노출. `_rnRequestTsa`는 `{ok,error}` 반환(위 5.2 반영).
- **⑥ Evidence Chain ID:** `EC-YYYYMMDD-NNNNNN` — 노트·PDF·git·TSA·출원을 한 사건번호로. **생성 시 고정 → 해시 포함**(불변). UUID가 진짜 키, EC는 사람용 라벨(다기기 병합 시 라벨 중복 가능하나 무해).
  ```js
  function _rnMakeEcId(existing) {
    const d = new Date(); const ymd = `${d.getFullYear()}${String(d.getMonth()+1).padStart(2,'0')}${String(d.getDate()).padStart(2,'0')}`;
    const seq = ((existing||[]).filter(e => e && e.evidence_chain_id).length + 1);
    return `EC-${ymd}-${String(seq).padStart(6,'0')}`;   // rnAddEntry에서 entry_hash 계산 전에 부여
  }
  ```
- **④ openssl 검증 명령 자동생성:** 상세에 "📋 복사" 버튼(clipboard→실패 시 .sh 다운로드) + MD에도 포함. 변리사·심사관이 그대로 붙여넣어 검증.
  ```js
  function _rnOpensslCmd(entry) {
    if (!entry.tsa_token) return '# (이 기록에는 TSA 토큰이 없습니다)';
    const hash = entry.tsa_request_hash || entry.entry_hash || '';
    return [
      `# Evidence Chain ID: ${entry.evidence_chain_id||''} · TSA: ${entry.tsa_provider||'?'}${entry.tsa_timestamp?' @ '+entry.tsa_timestamp:''}`,
      `# 1) TSA CA 인증서를 제공자 사이트에서 받아 tsa_ca.pem 로 저장`,
      `cat > note.tsr.b64 <<'TST'`, entry.tsa_token, `TST`,
      `base64 -d note.tsr.b64 > note.tsr`,
      `openssl ts -verify -digest ${hash} -in note.tsr -CAfile tsa_ca.pem`,
      `# → "Verification: OK" 이면 그 해시가 토큰 시각에 존재했음이 신뢰기관에 의해 증명됨.`,
    ].join('\n');
  }
  ```
- **③ 보류(정직):** PDF·git 해시 추가 타임스탬프는 실익 적음 — git 커밋은 GitHub가 이미 타임스탬프, PDF는 인쇄 생성이라 안정적 파일 바이트 없음.
